## SYSTEM OVERVIEW

    [Farmer's Phone]
         │
         ▼
    Take photo → TFLite embedding (1024 floats) → PCA(64) on server → SMS
         │
         ▼ (GSM / 2G — no internet needed)
    [SMS Gateway: Twilio]
         │
         ▼
    [Cloud Server: FastAPI + SQLite]
    Parse SMS → PCA transform → XGBoost classify → Tamil reply
         │
         ▼ (SMS back to farmer)
    [Farmer sees: disease name + action + confidence]
         │
         ▼
    Farmer replies YES/NO → logged in SQLite → retraining data

## STEP 2 — GET MOBILENET V3 TFLITE MODEL

    Option A — Python script (recommended):

        import tensorflow as tf
        import tensorflow_hub as hub

        model = tf.keras.Sequential([
            hub.KerasLayer(
                "https://tfhub.dev/google/imagenet/mobilenet_v3_small_100_224/feature_vector/5",
                trainable=False
            )
        ])
        model.build([None, 224, 224, 3])

        converter = tf.lite.TFLiteConverter.from_keras_model(model)
        tflite_model = converter.convert()

        with open("mobilenet_v3_small.tflite", "wb") as f:
            f.write(tflite_model)

        print("Saved mobilenet_v3_small.tflite")

    Option B — Direct download:
        wget https://storage.googleapis.com/tfhub-lite-models/google/lite-model/imagenet/mobilenet_v3_small_100_224/feature_vector/5/default/1.tflite \
             -O mobilenet_v3_small.tflite

    Place the .tflite file in:
        • Same directory as 1_train.py (for training)
        • app/src/main/assets/  (for Android app)

## STEP 4.1 — DEPLOY THE SERVER (Option B: Firebase Cloud Functions) - NEW

    If you prefer a serverless setup:
    1. Install Firebase CLI: `npm install -g firebase-tools`
    2. Login: `firebase login`
    3. Initialize: `firebase init functions` (Select Python)
    4. Copy files:
       - Copy `firebase_backend/main.py` -> `functions/main.py`
       - Copy `firebase_backend/requirements.txt` -> `functions/requirements.txt`
       - Copy `agriai_model.pkl`, `agriai_pca.pkl`, `agriai_labels.json` -> `functions/`
    5. Deploy: `firebase deploy --only functions`
    6. Set environment variables in the Firebase Console:
       - `AGRIAI_NUMBER`, `TWILIO_SID`, `TWILIO_TOKEN`, `WEBHOOK_SECRET`

    Firestore Configuration:
    - Create a Firestore database in the Firebase Console.
    - No collection setup needed; the function will create `pending`, `feedback`, and `rate_log` automatically.

## COMMON ERRORS AND FIXES

    Error: "TFLite model input shape mismatch"
    Fix:   Model expects [1, 224, 224, 3]. Check preprocess() output shape.

    Error: "Checksum mismatch" in server logs
    Fix:   SMS was truncated. Check message length — may need multipart SMS.

    Error: "Folder not found: Sugarcane_Red Rot"
    Fix:   Kaggle zip may have slightly different spacing. Check exact folder name.

    Error: Low accuracy (< 70%) after training
    Fix:   Check that TFLITE_MODEL is the same file used in training AND in the app.
           Different TFLite versions produce different embeddings.

    Error: App crashes on older phones
    Fix:   Some phones have < 2GB RAM. Reduce IMG_SIZE to 160×160 in both
           TFLiteHelper.kt and 1_train.py — retrain the model after.

    Error: "agriai_pca.pkl not found" on server
    Fix:   Copy agriai_pca.pkl alongside model.pkl on the server.
           Server will work without it but won't apply PCA transform.

    Error: "no such table: pending" on server
    Fix:   Delete agriai_server.db and restart — tables are auto-created.
